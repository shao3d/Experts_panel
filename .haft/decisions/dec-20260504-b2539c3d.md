# V1 thin authenticated REST context endpoint with explicit-only subagent and source_bundle partial pipeline

## 1. Problem Frame

**Signal:** The system already has an internal research pipeline (`POST /api/v1/query`, expert Telegram corpus, Reddit sidecar, citation-oriented synthesis, and the AI Architect concept), but the intended new use is external AI agents requesting context/data for trends, software, approaches, and analysis. The mismatch is that the current surface is UI/SSE-oriented and not yet a stable, safe, agent-facing contract.

**Constraints:**
- Do not break the existing `POST /api/v1/query` UI/SSE workflow.
- Do not expose secrets, raw credentials, admin operations, imports, or maintenance endpoints to external agents.
- External access must have authentication, rate limits, timeout/cost budgets, request IDs, and audit logging before production exposure.
- Returned answers/context must preserve source references and confidence/abstention behavior; no source-free market claims as the default contract.
- The first step must be a narrow MVP surface, not a full platform, marketplace, or mass rewrite of the retrieval pipeline.

**Acceptance:** A bounded first external surface exists that AI agents can call safely for 3-5 representative research/context queries, returns structured source-backed context, has explicit auth/rate/cost controls, does not expose admin/data-management operations, and does not regress the existing UI/SSE query workflow.

## 2. Decision

**Selected:** V1 thin authenticated REST context endpoint with explicit-only subagent and source_bundle partial pipeline

**Selection policy:** Pick the smallest reversible agent-facing surface that validates utility while preserving existing UI/SSE behavior, keeping calls explicit-only, avoiding unnecessary synthesis cost, and preserving source/comment evidence.

**Why selected:** The chosen direction gives Codex/Claude Code an explicit manual way to query Experts Panel without automatic background calls, reuses the existing retrieval relevance pipeline, returns source-backed evidence packets instead of full UI answers, supports all/custom expert selection through `expert_filter`, and keeps MCP as a later adapter after the REST/source_bundle contract is proven.


**Invariants:**
- Main Codex/Claude Code agents must not query Experts Panel automatically; access is explicit-only by user request or slash command.
- Existing `POST /api/v1/query` UI/SSE behavior must remain unchanged.
- The new agent endpoint is read-only and must not expose admin/import/maintenance operations.
- Default subagent response mode is `source_bundle`, not full prose answer.
- For source_bundle mode, return the sources selected by the system under the query scope; do not dump all posts for selected experts.
- Comments under selected main sources are included; drift comment groups and comment synthesis are disabled by default.
- Expert scope must be visible in the response as `selection_used`.
- Auth, rate/cost controls, timeouts, request ids, and audit logging are required before production exposure.

**Pre-conditions:**
- [ ] Create an architecture spec before implementation.
- [ ] Identify current source-selection seam because existing `main_sources` are reduce-derived.
- [ ] Define explicit request/response schemas including expert selection and source_bundle behavior.
- [ ] Define Fly.io deployment constraints and cost/latency gates.

**Post-conditions:**
- [ ] `POST /api/v1/agent/context` or equivalent returns source_bundle packets for explicit subagent calls.
- [ ] Subagent instructions enforce explicit-only usage and expert-scope parsing.
- [ ] Source bundle includes selected sources plus comments under selected sources, with pipeline_used and pipeline_skipped metadata.
- [ ] No drift group scoring, comment synthesis, reduce answer synthesis, or meta synthesis runs in default source_bundle mode.
- [ ] Existing UI tests/routes continue to work.

**Admissibility:**
- NOT: Anonymous public access.
- NOT: Automatic background calls from main agents.
- NOT: Dumping entire expert corpora to Codex.
- NOT: Silently broadening expert selection.
- NOT: Running the full UI answer pipeline for default source_bundle mode.
- NOT: Exposing secrets or admin/data mutation endpoints.

## 3. Rationale

**Counterargument:** A full MCP tool or full synthesis endpoint might feel more natural to agents and avoid adding another REST surface, but it increases first-step complexity and risks automatic/over-broad usage before we understand the useful context contract.

**Selected variant weakest link:** The current `main_sources` are produced inside `ReduceService`; source_bundle mode must extract or recreate source selection before full answer synthesis without regressing existing `/api/v1/query`.

**Rejected alternatives:**
| Variant | Verdict | Reason |
|---------|---------|--------|
| V1 thin authenticated REST context endpoint with explicit-only subagent and source_bundle partial pipeline | **Selected** | The chosen direction gives Codex/Claude Code an explicit ... |
| Remote MCP server first | Rejected | Better agent-native packaging eventually, but too much protocol/security surface before the source_bundle contract is proven. |
| Private local bridge only | Rejected | Good for dogfooding but insufficient as the durable API contract. |
| Full AI Architect API | Rejected | Too broad for first validation and couples many product risks. |
| Source bundle API as a separate product | Rejected | Best folded into V1 as the required default response mode for subagent usage. |

**Evidence requirements:**
- Unit/API tests for expert_filter all/custom/group mapping.
- Tests or contract checks showing source_bundle skips reduce/comment synthesis/drift scoring by default.
- A request fixture showing comments under selected main sources are returned.
- Auth/rate-limit/cost-control verification before production exposure.
- Fly.io smoke test or documented local-only gate before live use.

**Predictions:**
| Claim | Observable | Threshold |
|-------|------------|-----------|
| Partial source_bundle pipeline will materially reduce latency/cost compared with full `/api/v1/query` for Codex research calls. | Same query/custom expert set run through full UI endpoint vs agent source_bundle endpoint. | source_bundle avoids Reduce, language validation, drift scoring, comment synthesis, and meta synthesis by default; elapsed time and LLM calls are lower in instrumentation. |
| Explicit-only subagent prevents accidental Experts Panel usage in normal coding chats. | Subagent instructions and command invocation logs. | No Experts Panel requests occur unless user prompt contains explicit trigger or slash command. |
| Returning comments under selected sources improves Codex usefulness without enabling drift scoring. | Source_bundle response examples and user evaluation. | Responses include author/community comments for main_sources and no drift groups unless opt-in. |

## 4. Consequences

**Rollback plan:**
Triggers:
- Agent endpoint causes cost spikes, accidental calls, or source selection quality regressions.
- Endpoint breaks existing UI/SSE pipeline.
- Auth/rate limiting cannot be implemented safely before exposure.
Steps:
1. Disable the agent endpoint route or token.
2. Keep `/api/v1/query` unchanged.
3. Remove subagent/command access until contract is fixed.
Blast radius: New agent API and subagent integration only; existing UI/query path should remain unaffected.

**Refresh triggers:**
- If MCP remote tool support becomes the primary target integration path.
- If external usage needs streaming progress rather than request/response packets.
- If Fly.io costs, cold starts, or memory pressure make production endpoint unreliable.
- If source selection quality is poor without ReduceService.
- If user wants automatic background research rather than explicit-only calls.
