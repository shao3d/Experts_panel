# AND-21 Panex product-quality eval starts as deterministic guardrail plus human review

## Rationale

We need to evaluate final Панэкс answer usefulness separately from API contract correctness. A fully automated LLM-as-judge would create false certainty at this stage, while pure manual review would be hard to repeat. The chosen first slice creates golden scenarios, a human-readable rubric, and deterministic checks for stable properties such as Request passport, source handles, signal honesty, answer coverage, actionability, brevity, expansion path, and external-link boundaries. Human review remains the final product-quality judge for the first scenarios.
