# Close MVP open questions for agent context API

## Rationale

Accepted three implementation decisions for the first Experts Panel agent-facing surface: attach linked CONTEXT only through explicit resolve provenance, use a local CLI wrapper before production Fly exposure, and keep the first experts_panel_researcher subagent repo-local before any global user-level shortcut. Alternatives lost because heuristic CONTEXT attachment can misattribute evidence, direct production Fly usage increases cost/latency/token exposure before the contract is proven, and a global subagent widens blast radius while the API is still evolving.

## Evidence

User accepted the proposed choices in chat on 2026-05-04; spec updated to reflect them in source selection, subagent/wrapper roadmap, acceptance criteria, and closed decisions.
