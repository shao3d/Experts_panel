# Force Embs&Keys retrieval for Agent Context calls

## Rationale

Agent Context is an explicit subagent/API surface, not the interactive UI. The UI can still toggle Embs&Keys search for /api/v1/query, but subagent/CLI source_bundle calls must consistently use embedding-backed hybrid retrieval so results are not dependent on frontend state or user UI preferences. CLI sends use_super_passport=true, the API normalizes selection_used.use_super_passport=true even if a caller sends false, and AgentContextService prepares one query embedding for all selected experts before bounded parallel expert processing.

## Evidence

Targeted tests assert CLI payload includes use_super_passport=true, API normalizes false to selection_used true, and AgentContextService passes the precomputed query embedding into HybridRetrievalService for each selected expert.
