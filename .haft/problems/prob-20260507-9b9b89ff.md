# Next Panex product-value upgrades after AND-23 selector expansion

## Signal

AND-23 production dogfood proved that Панэкс can answer a digest question on Fly.io and then handle natural Russian follow-ups through exact source_expand without source_key jargon. The next product-value problem is choosing what improves real day-to-day use from other repos now: portability, continuity, decision usefulness, long-run ergonomics, or external evidence depth, without breaking explicit-only behavior or turning Панэкс into broad web search.

## Problem Type

optimization

## Constraints

- Панэкс remains explicit-only; no automatic calls from Codex/Claude by default.
- Default external links remain author-supplied references unless a future mode is explicitly requested.
- Do not add a hidden full UI Reduce/Meta/Reddit pipeline to agent-facing calls.
- Keep default answers compact, source-backed, and practitioner-opinion framed.
- Preserve exact source handles and source_expand audit path.
- Respect Fly.io latency/cost constraints and long-running nature of all-expert calls.
- Do not require the user to know API jargon such as source_key/source_expand.

## Optimization Targets

- Increase day-to-day usefulness from other repos
- Reduce friction to invoke Панэкс correctly outside Experts_panel repo
- Improve continuity and auditability across digest -> expansion workflows

## Observation Indicators

- Extra latency/cost from new LLM calls or long all-expert jobs
- User confusion from too many modes or commands
- Security/provenance risk from external fetching
- Prompt/template sprawl
- State/privacy complexity from result persistence

## Acceptance

Produce a ranked set of next upgrade variants after AND-23 with clear product value, risk, implementation effort, and a recommended next slice. The recommendation should preserve explicit-only invocation, production Fly.io target, source provenance, compact answers, and cross-repo usability.

## Blast Radius

User-level/repo-local Панэкс agent, CLI/wrapper portability, optional backend API/storage endpoints, docs, product-quality tests, production Fly dogfood.
