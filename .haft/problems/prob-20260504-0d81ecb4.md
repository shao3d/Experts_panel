# Expose Experts Panel as an agent-facing research/context API

## Signal

The system already has an internal research pipeline (`POST /api/v1/query`, expert Telegram corpus, Reddit sidecar, citation-oriented synthesis, and the AI Architect concept), but the intended new use is external AI agents requesting context/data for trends, software, approaches, and analysis. The mismatch is that the current surface is UI/SSE-oriented and not yet a stable, safe, agent-facing contract.

## Problem Type

synthesis

## Constraints

- Do not break the existing `POST /api/v1/query` UI/SSE workflow.
- Do not expose secrets, raw credentials, admin operations, imports, or maintenance endpoints to external agents.
- External access must have authentication, rate limits, timeout/cost budgets, request IDs, and audit logging before production exposure.
- Returned answers/context must preserve source references and confidence/abstention behavior; no source-free market claims as the default contract.
- The first step must be a narrow MVP surface, not a full platform, marketplace, or mass rewrite of the retrieval pipeline.

## Optimization Targets

- agent usability and integration fit
- evidence quality and source traceability
- small reversible implementation scope

## Observation Indicators

- latency and LLM spend per agent request
- source mismatch or hallucination incidents
- support/ops burden from external consumers
- whether agents actually reuse returned context instead of asking for broad prose

## Acceptance

A bounded first external surface exists that AI agents can call safely for 3-5 representative research/context queries, returns structured source-backed context, has explicit auth/rate/cost controls, does not expose admin/data-management operations, and does not regress the existing UI/SSE query workflow.

## Blast Radius

Backend API contract, retrieval/synthesis pipeline, auth and deployment surface, external-agent integrations, cost controls, and trust in the system's research output.

## Reversibility

medium

**Parity plan:**
- Baseline set: V1, V2, V3, V4
- Window: 2026-Q2 first agent-facing MVP planning; no production implementation started yet
- Budget: First useful slice should be scoped to days, not weeks: reuse existing query pipeline and avoid broad data-model rewrites.
- Missing data policy: explicit_abstain
- Pinned condition: All variants must preserve existing UI/SSE behavior.
- Pinned condition: All variants must rely on the existing Experts Panel corpus/pipeline for the first step.
- Pinned condition: All variants are evaluated for a first MVP, not the final long-term platform.
- Pinned condition: No variant may assume public anonymous access without authentication/cost controls.
- Pinned condition: Compare variants only as first MVP approaches for the same idea: external AI agents requesting context for trends, tools, approaches, and analysis. Do not score them as full future platform end states.
