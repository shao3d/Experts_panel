# Next Panex product-value upgrades after AND-22 dogfood

## Signal

AND-22 production dogfood showed Панэкс is already useful as an explicit-only research helper: compact answers, weak-signal honesty, human follow-up source expansion, external-link boundary, and exact scope discipline all passed. The next problem is choosing upgrades that increase day-to-day product value in other repos without turning Панэкс into automatic web search, an overbuilt platform, or a hidden second synthesis pipeline.

## Problem Type

optimization

## Constraints

- Панэкс remains explicit-only; no automatic calls from Codex/Claude by default.
- Default external links stay references-only unless a future mode is explicitly requested.
- Do not add a broad always-on pipeline or hidden full UI Reduce/Meta synthesis.
- Keep default answers compact and source-backed.
- Preserve source handles/source_expand as the audit path.
- Respect Fly.io latency/cost constraints for long expert runs.

## Optimization Targets

- Increase day-to-day usefulness in other repos
- Improve follow-up ergonomics after digest answers
- Preserve trust/calibration and source provenance

## Observation Indicators

- Extra latency/cost from new LLM calls
- User confusion from too many modes/terms
- Prompt/template sprawl
- External-fetch security/provenance risk

## Acceptance

Produce a ranked set of next upgrade variants with clear product value, implementation risk, and recommended next slice. The recommendation should preserve explicit-only use, source provenance, compact answers, and production Fly.io practicality.

## Blast Radius

Agent-facing API, Панэкс subagent behavior, product-quality tests/docs, optional backend support endpoints.

## Reversibility

medium
