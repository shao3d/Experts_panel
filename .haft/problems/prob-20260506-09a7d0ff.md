# Next product-value upgrades for Experts Panel agent-facing API

## Signal

Experts Panel now has a production explicit-only agent-facing API with expert_digest and source_expand. The next question is which upgrade increases practical value for Codex/Claude Code users without turning the API into automatic web search, a second digest pipeline, or an overbuilt platform.

## Problem Type

synthesis

## Constraints

- Keep explicit-only invocation; no automatic Панэкс calls for generic tasks.
- Keep practitioner posts framed as source-backed opinions, not proven facts.
- Avoid reintroducing the full UI Reduce/Meta/Comment synthesis pipeline by default.
- Keep Панэкс user language human-readable; users should not need API jargon such as source_key/source_expand.
- Respect Fly.io production constraints: long all-expert requests are natural, but default UX should not require repeated expensive runs.
- Do not fetch/crawl external links unless a future mode is explicitly requested.

## Optimization Targets

- Higher day-to-day usefulness for AI coding/research work in other repos
- Better trust/calibration of expert evidence
- Low implementation risk and no hidden pipeline creep

## Observation Indicators

- LLM/API cost and latency
- Response size and parent-agent context pressure
- Risk of agent overclaiming practitioner opinions

## Acceptance

Produce a ranked set of next upgrade variants with clear product value, implementation cost/risk, and recommended next slice.

## Blast Radius

Agent-facing API, Панэкс subagent contract, docs/tests, and possibly backend service/request models.

## Reversibility

medium
