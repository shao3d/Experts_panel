Три уровня ценности для SODA

Что мы получили и зачем это SODA.

Предыстория
В SODA может быть запоминающийся персонаж(и) или маскот(ы), который «живёт» на экране пользователя и визуально отражает работу AI-консультанта или другие манипуляции пользователя на экране. pixel-agents — готовая база для этого: персонажи с поведением, анимациями и навигацией. Мы проверили движок на реальном продакшне и задокументировали перенос.
Скорость

Pixel-офис в SODA — за один день

Движок вычищен, адаптирован под браузер, покрыт тестами. Не нужно разбираться с оригинальным репо. Копируем 17 файлов, пишем ~310 строк обвязки, подключаем к своему пайплайну.

Масштаб

Рецепт трансфера — для любого фронтенда

Задокументирован каждый шаг: что копировать, что писать с нуля, в каком порядке инициализировать, какие параметры менять. Porting Guide — 563 строки + чеклист из 11 пунктов. Разработчик повторит по нему, не изобретая заново.

Wow-фактор

Персонажи реагируют на AI в реалтайме

Когда AI ищет информацию — персонажи читают. Когда синтезирует ответ — печатают. При смене фазы — встают, идут к другому столу. Пользователь видит конкретную работу, а не абстрактный progress bar.

Что умеет движок

Оригинал — VS Code extension (pixel-agents, MIT лицензия). Мы вытащили движок из редактора, адаптировали под браузер и подключили к React. Ноль npm-зависимостей, ~100 KB ассетов.

  • 🏢
    Тайловый офис. Сетка произвольного размера. Стены, полы с 9 паттернами, мебель — всё из PNG-спрайтов.
  • 🧑‍💻
    Персонажи с поведением. Конечный автомат: печатают, читают, ходят, отдыхают. Навигация по офису с обходом препятствий. 6 уникальных палитр + автоматический hue shift для 7-го и далее. До 10 одновременно.
  • 🎨
    Динамическая раскраска. Полы и мебель красятся в runtime через HSL — один набор серых спрайтов, разные цветовые схемы без дополнительных ассетов.
  • 🪑
    30+ предметов мебели. Столы, стулья, ПК (с анимацией экранов), диваны, растения, картины, книжные шкафы. Ротация и зеркалирование автоматические.
  • 📐
    Z-сортировка. Персонажи корректно проходят за и перед мебелью — как в изометрических играх.
  • ✨
    Matrix-эффект. При появлении и исчезновении персонажей — эффект цифрового дождя (0.3 сек). Визуально отмечает начало и конец работы агента.
  • 📱
    Только десктоп. На мобильных движок не загружается вообще — code splitting через React.lazy. Ноль лишнего трафика.

Anti-Drone System

Как превратить строй роботов в живой офис, где каждый персонаж ведёт себя индивидуально.

Главная проблема: AI-пайплайн переключает фазы — и все персонажи синхронно дёргаются. Выглядит как марш роботов. Решение — три уровня «оживления»:

Фаза пайплайна
Поиск, анализ,
синтез...
→
Пропорции
40% печатают,
60% читают
→
Рандом
Кто именно —
случайный выбор
→
Каскад
Переключаются
один за другим
  • ⚖️
    Пропорциональное распределение. Если 2 фазы «синтез» и 3 фазы «поиск» активны — 40% персонажей печатают, 60% читают. Не «все делают одно».
  • 🎲
    Fisher-Yates перемешивание. Кто именно печатает, а кто читает — определяется случайно при каждой смене фазы. Нет «главного» персонажа.
  • 🌊
    Каскадный стаггер. Персонажи переключаются не одновременно, а волной — с задержкой 0.5–1.2 секунды между каждым. Визуально — плавная волна, а не синхронный рывок.
  • 🪑
    Ротация мест. «Читатели» уходят в лаунж-зоны (кухня, библиотека). «Писатели» садятся за ПК. При смене фазы — видимое перемещение по офису.
  • 🎚️
    20%-порог. Мелкие колебания пайплайна (38% → 42%) не перетриггеривают пересортировку. Только значимые изменения вызывают движение.

Весь Anti-Drone реализован в React-обвязке, а не в движке. Это значит: движок можно использовать как есть, а «оживление» адаптировать под логику конкретного проекта.

Три слоя переноса

Движок копируем. Ассеты качаем скриптом. Интеграцию пишем под свой проект.

1

Движок — копируем без изменений

17 файлов TypeScript, 4 477 строк. Рендерер, персонажи, мебель, тайлы, загрузчик ассетов. Просто скопировать в новый проект.

Время: 5 минут
2

Ассеты — скрипт автоматизирует

~100 PNG-файлов: персонажи, полы, стены, мебель. Готовый bash-скрипт клонирует pixel-agents и раскладывает всё по папкам. Один запуск.

Время: 1 минута
3

Интеграция — пишем под SODA

~310 строк: React-обвязка, маппинг фаз на анимации, media query для мобильных. API движка — всего 6 методов (добавить/убрать персонажа, активировать, задать действие, пересадить). Здесь адаптируем маппинг фаз под логику SODA.

Время: 4–6 часов

Итого: 1 рабочий день

От пустого проекта до работающего пиксельного офиса с персонажами, которые реагируют на AI-пайплайн. Все грабли уже задокументированы.

9 подводных камней

Каждый стоил бы часы отладки. Теперь они задокументированы — и не повторятся.

P1

Zoom минимум 3x

При меньшем зуме пиксели размываются. Canvas рисует крупно, CSS масштабирует вниз.

P2

Прозрачность ≤ 2

Порог альфа-канала — не ноль, а 2. Иначе артефакты по краям всех спрайтов.

P3

Порядок инициализации

Полы → стены → мебельный каталог → персонажи → офис. Нарушишь — стулья пропадут.

P4

ID — только числа

Движок принимает number, не string. Строковые ID хешируются в числа.

P5

Два файла констант

Оригинал хранит константы в двух местах. Мы объединили в один — при копировании из оригинала нужно повторить.

P6

React 18 vs 19

Оригинал на React 19. useRef в React 18 требует null. Мелочь, но ломает TypeScript.

P7

Sitting offset ≠ z-sort

Смещение сидящего персонажа — только визуальное. Если применить к z-сортировке — провалится сквозь стол.

P8

OfficeState — не рефакторить

852 строки god-object. Выглядит страшно, но работает. Менять только входы и выходы.

P9

Загрузка ассетов — 60% работы

Движок скопировать легко. Основное время — browserAssetLoader, загрузка PNG в браузере.

4 дня: от ресёрча до продакшна

Хронология работы — что было сделано и когда.

28 марта
Ресёрч и планирование
Изучение pixel-agents: архитектура, спрайты, движок. Определение стратегии переноса. Что копировать, что писать с нуля, что выбросить (WYSIWYG-редактор).
29 марта
Прототип и браузерный загрузчик
Ключевая проблема: оригинал работал внутри VS Code и имел прямой доступ к файлам. В браузере этого нет. Написан browserAssetLoader (415 строк) — загружает и декодирует все PNG-спрайты через Canvas API. Скрипт копирования ассетов.
30 марта
Первый запуск + интеграция с пайплайном
Движок заработал в Canvas. React-обвязка. Подключение к SSE-стриму. Персонажи начали реагировать на фазы AI. Маскот для мобильных. 55 тестов написаны и пройдены.
31 марта
Продакшн-полировка
Расширение до 4 комнат (42×15). Anti-Drone System: пропорциональное распределение + каскадный стаггер + ротация мест. Тёплая цветовая палитра. Документация Porting Guide. Деплой в прод.

12 коммитов за 2 дня активной разработки (30–31 марта). Ресёрч и прототипирование — ещё 2 дня (28–29 марта).
